# HAINOA Site

Version: v0.2  
Status: Active architecture site

This repository contains the public website for **HAINOA**.

**HAINOA** stands for **HAIN Open Architecture**.  
It is the architecture layer within the HAIN system and provides the structural language, interpretive logic, and reference direction beneath the broader Hybrid AI Native category.

## Role in the HAIN system

Within the current HAIN structure:

- **HAIN World** introduces the system
- **HAINATIVE** defines the category
- **HAINOA** explains the architecture
- **HAINPRO** maps and specifies the protocol layer
- **HAINIZE** operationalizes the system

This repository belongs to the third layer: the architecture layer.

## What this repository contains

This repository currently contains:

- the public architecture-facing site for **HAINOA**
- explanatory content around the HAIN system structure
- principles and reference direction for Hybrid AI Native architecture
- links to the whitepaper, glossary, HAIN World, HAINATIVE, HAINPRO, and HAINIZE
- site assets such as favicon and OG image
- the domain configuration for `hainoa.org`

## What HAINOA is for

HAINOA exists to make the HAIN system structurally legible.

Its purpose is to provide:

- a clearer architectural language
- a structured explanation of system layers
- reference direction across cloud, edge, private, and agentic environments
- a stable interpretive layer between category definition and protocol specification

HAINOA is not intended to act as a product surface.  
It is the architecture layer that gives the HAIN system structural coherence.
